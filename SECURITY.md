# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest (main branch) | Yes |
| Older releases | Best effort |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly. **Do not open a public issue for security vulnerabilities.**

### How to Report

1. **Email:** Send a detailed report to the repository owner via the contact information on their [GitHub profile](https://github.com/E339-art).
2. **Include:** A description of the vulnerability, steps to reproduce it, potential impact, and any suggested fixes.
3. **Response Time:** You can expect an initial response within 48 hours acknowledging receipt of the report.

### What to Expect

- Acknowledgment of your report within 48 hours
- An assessment of the vulnerability and its severity within 5 business days
- A timeline for a fix, if applicable
- Credit in the release notes (unless you prefer to remain anonymous)

### Scope

This policy applies to the code and configurations maintained in this repository. Third-party dependencies are managed through regular updates and automated vulnerability scanning.

## Security Best Practices

This project follows these security practices:

- **Dependency Management:** Regular updates and automated vulnerability scanning via GitHub Dependabot
- **Secret Management:** No secrets or credentials are committed to the repository
- **Code Review:** All changes undergo review before merging
- **CI/CD Security:** Automated security checks run on every pull request

## Acknowledgments

We appreciate the security research community and thank all individuals who responsibly disclose vulnerabilities.
